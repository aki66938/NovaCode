# NovaCode 维护者名单

> 本文件是 NovaCode 项目**权限的唯一事实源**。谁拥有合并 `dev → main`、打 tag、发版、裁决版本号的权力，以本文件当前内容为准。
> 任何权限变更（增设、移除、转移）都必须以一次对本文件的提交记录在案，并由现任主创批准。

## 主创（Lead Maintainer）

主创对版本号、发布、架构方向、治理规则拥有最终裁决权。

| 角色 | GitHub | 职责 |
|------|--------|------|
| 主创 | [@aki66938](https://github.com/aki66938) | 版本/tag/发版裁决、架构方向、治理与维护者任免 |

## 维护者（Maintainers）

维护者可 review 并合并 PR 进 `dev`，但不得单独改动版本线或执行 `dev → main` 发版（除非主创授权）。

| 角色 | GitHub | 范围 |
|------|--------|------|
| —（暂无） | — | — |

## 变更流程

1. 现任主创在 PR 中修改本文件（增/删/转移角色）。
2. 主创合并该 PR 即生效。
3. 主创转让：在本文件中更新主创条目，并在 PR 描述中明确交接。
4. 主创长期缺席且未指定继任时，按 [GOVERNANCE.md](GOVERNANCE.md) 的继任条款处理。

## Fork 与独立分支

任何人 fork 本仓库进行二次开发后，即成为其自有 fork 的主创，自行维护其 `MAINTAINERS.md`；但须遵守 [LICENSE](LICENSE) 的出处署名条款，保留 `Based on NovaCode by aki66938`。
